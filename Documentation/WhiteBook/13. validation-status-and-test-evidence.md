# Validation Status and Test Evidence

## Goal of this chapter

This chapter summarizes the current validation landscape from the live repository, identifies which architectural claims are directly supported by named test suites, and explicitly avoids unverified pass-count claims.

- `365` `.cs` source files under `HybridCPU_ISE/`

These are live-tree counts, not claims about a fresh repository-wide pass total.
They are backed by `build/recount-validation-evidence.ps1` and the claim-to-suite map in `Documentation/evidence-matrix.md`.

The suite names show active coverage over at least the following architectural areas:

- canonical decode and ISA surface
- retired compatibility boundaries
- pipeline FSM and execution follow-through
- typed-slot scheduler and SMT densification behavior
- legality service and guard matrices
- replay certificate and replay-token behavior
- diagnostics, tracing, and telemetry truth
- CSR and VMX surfaces
- memory subsystem and binding seams
- architectural audit and fail-closed boundaries

## Why older pass counts are not repeated here

The existing WhiteBook README mentions a "305 passing tests" baseline. That number is not repeated in this chapter because it was not revalidated against the current tree with a fresh full-suite execution during this pass.

This chapter intentionally prefers:

- named suites and their architectural relevance;
- explicit gaps in runtime verification capability from the current environment;
- no stale totals presented as current truth.
