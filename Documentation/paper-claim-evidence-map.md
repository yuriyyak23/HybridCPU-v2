# Paper Claim Evidence Map

This document is a reviewer aid for the current manuscript revision. It does
not outrank live code: live code wins, and the docs/tests listed below only
bound how repository-facing claims may be stated.

Detailed manuscript-term to implementation-anchor crosswalks live in
`Documentation/manuscript-implementation-anchor-map.md`. The present document
stays at the claim-family level so that the submission-facing manuscript can
remain architecture-first while the repository keeps its code-faithful
traceability.

The safe reading rule is simple:

- use manuscript sections for argument flow;
- use the boundary docs above for repository-facing terminology;
- use the implementation-anchor map when exact code surfaces are needed;
- refuse any stronger claim once it crosses the stated current bounded limit.
